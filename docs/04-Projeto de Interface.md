
# Projeto de Interface

<span style="color:red">Pré-requisitos: <a href="2-Especificação do Projeto.md"> Documentação de Especificação</a></span>

O projeto de interface do sistema MEI Easy foi desenvolvido com o objetivo de estruturar a experiência do usuário, abrangendo desde o fluxo de navegação até a representação visual das telas da aplicação. Inicialmente, foi elaborado um diagrama de fluxo, com a finalidade de mapear a interação do usuário com o sistema, identificando as principais ações, decisões e caminhos possíveis, priorizando a lógica de uso em detrimento do design visual.

Na sequência, foram desenvolvidos os wireframes das principais telas do aplicativo mobile, utilizando a ferramenta Figma. Esses protótipos foram construídos em baixa fidelidade, focando na organização dos elementos de interface e na navegação entre as telas, contemplando funcionalidades como login, cadastro, dashboard, registro de movimentações, relatórios, perfil e metas.

A definição das interfaces foi baseada nos requisitos funcionais, não funcionais e nas histórias de usuário previamente levantadas, garantindo que a aplicação atenda às necessidades do público-alvo. Além disso, o protótipo foi estruturado de forma interativa, permitindo simular a navegação do usuário e validar a usabilidade da solução antes da etapa de desenvolvimento.

## Diagrama de Fluxo

O diagrama apresenta o estudo do fluxo de interação do usuário com o sistema interativo e  muitas vezes sem a necessidade do desenho do design das telas da interface. Isso permite que o design das interações seja bem planejado e gere impacto na qualidade no design do wireframe interativo que será desenvolvido logo em seguida.
